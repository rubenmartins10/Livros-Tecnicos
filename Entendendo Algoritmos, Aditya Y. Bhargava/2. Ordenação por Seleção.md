- Cada vez que quero armazenar um item na memória, peço ao computador um pouco de espaço e ele dá-me um endereço no qual eu posso armazenar o item. Se quero armazenar múltiplos itens, existe duas maneiras de o fazer : **arrays e listas**.

- Um **array** significa que todas as tarefas estão armazenadas umas ao lado das outras na memória.

- Podemos solicitar ao computador por 10 espaços, só por via de dúvidas. Então posso adicionar 10 itens à lista sem precisar de mover nada. Isto é uma boa maneira de controlar o problema de eventualmente precisasse de colocar outro item no array e ter de mover tudo por falta de espaço, mas atenção às desvantagens:
  - Poderei não precisar dos espaços extra que reservei; então a memória é desperdiçada. Não estou a utilizar a memória que solicitei, mas também mais ninguém a poderá utilizar.
  - Poderei ter de adicionar mais de 10 itens à lista de tarefas, e então terei de mover os itens de qualquer das maneiras.

- Com as **listas encadeadas**, os itens podem estar em qualquer lugar da memória. 

- Cada item armazena o endereço do próximo item da lista.

- Adicionar um item a uma lista encadeada é fácil : coloco-o em qualquer lugar de memória e armazeno o endereço do item anterior.

- Listas encadeadas são ótimas se quiser ler todos os itens, mas um de cada vez. Se quiser ler um item e passar para outro, as listas encadeadas são **terríveis**.

- **Arrays** são ótimos para ler elementos aleatórios, pois encontra-se qualquer elemento instantaneamente num array. Numa lista encadeada, os elementos não estão próximos uns dos outros, então não dá para calcular instantaneamente a posição de um elemento na memória.

- A posição de um elemento é chamada de **índice**.

- E se quiser deletar (apagar) um elemento? É mais fácil fazê-lo utilizando listas encadeadas, pois é necessário mudar apenas o endereço para qual o elemento anterior está a apontar. Com arrays, precisa de se mover tudo quando um elemento é eliminado.

- Ao contrário do que ocorre com as inserções, a eliminação de elementos sempre funcionará. A inserção poderá falhar quando não houver **espaço suficiente na memória**.

- As inserções e eliminações terão tempo de execução O(1) somente se puder acessar instantaneamente o elemento a ser deletado.

- Os arrays são mais comuns porque permitem acesso aleatório. Existem 2 tipos de acesso: o **aleatório e o sequencial**. O sequencial significa ler os elementos um por um a começar no primeiro. 
  - Listas encadeadas só podem lidar com o acesso sequencial.

- A **ordenação por seleção** é um algoritmo bom, mas não muito rápido. O **Quicksort** é um algoritmo de ordenação mais rápido, que tem tempo de execução de apenas O(n log n).

- Recapitulando:
  - Quando se quer armazenar múltiplos elementos, usa-se um **array** ou uma **lista**.
  
  - No **array**, todos os elementos são armazenados ao lado um do outro.
  
  - Na **lista**, todos os elementos estão espalhados e um elemento armazena o endereço do elemento seguinte. 

  - Arrays permitem leituras mais rápidas.

  - Listas encadeadas permitem rápidas inserções e eliminações.

  - Todos os elementos de um array devem de ser do mesmo tipo.
