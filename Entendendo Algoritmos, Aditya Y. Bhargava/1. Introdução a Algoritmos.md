- Um algoritmo é um conjunto de instruções que realizam uma tarefa.

- A pesquisa binária é um algoritmo. A sua entrada é uma lista ordenada de elementos.

- Se o elemento que estou à procura estiver na lista, a pesquisa binária retorna a sua localização, caso contrário, a pesquisa binária retorna None.

- Para uma lista de **n** números, a pesquisa binária precisa de **log n** para retornar o valor correto, enquanto a pesquisa simples precisa de **n** etapas.

- A pesquisa binária só funciona quando a lista está ordenada.

- O número máximo de tentativas é igual ao tamanho da lista. A isso chamamos **tempo linear**.

- A pesquisa binária é executada com **tempo logarítmico**.

- A notação Big O permite comparar o número de operações.
    - Big O -> O(n), sendo O(n) o número de operações

- 5 tempos Big O, do mais rápido para o mais lento :
  - O (log n), também conhecido como tempo logarítmico. **Pesquisa Binária**.
  - O (n), conhecido como tempo linear. **Pesquisa Simples**.
  - O (n * log n), um algoritmo rápido de ordenação. **Ordenação Quicksort**.
  - O (n^2), um algoritmo lento de ordenação. **Ordenação por Seleção**.
  - O (n!), um algoritmo bastante lento.

- Não posso converter um tempo de execução na notação Big O para um número de operações, mas por enquanto é suficiente. Os principais pontos do capítulo são os seguintes:
  - A rapidez de um algoritmo não é medida em segundos, mas pelo crescimento do número de operações.
  - Discutimos sobre o quão rapidamente o tempo de execução de um algoritmo aumenta conforme o número de elementos aumenta.
  - O tempo de execução em algoritmos é expresso na notação Big O.
  - O (log n) é mais **rápido** do que O (n), e O (log n) fica ainda mais rápido conforme a lista aumenta.
